# Modern Vue Study Plan

Учебный план по современному `Vue 3`, составленный по мотивам структуры `RS School React`, но адаптированный под более глубокое и спокойное изучение без жесткой привязки к короткому таймлайну.

## Цель

Дойти до уверенного практического уровня в:

- `Vue 3`
- `Composition API`
- `TypeScript`
- `Vite`
- `Vue Router`
- `Pinia`
- `Vitest`
- `Vue Test Utils`
- `SSR / SSG`
- `Nuxt 3`

Этот план подходит разработчику, который уже знает основы `JavaScript`, умеет пользоваться `Git` и понимает, как работают `HTML`, `CSS` и `REST API`.

## Как проходить план

- Не спешить: важнее глубина понимания, чем скорость прохождения.
- На каждый модуль делать хотя бы одну законченную практическую работу.
- После каждых 2-3 модулей возвращаться и делать рефакторинг старых задач.
- Теорию изучать по официальной документации, русскоязычные видео использовать как поддержку.
- Все примеры писать сразу на `TypeScript`.
- Для каждого модуля фиксировать критерии завершения: что именно должно быть реализовано, проверено и объяснимо своими словами.

## Как понимать, что модуль завершен

Модуль можно считать завершенным, если:

- сделана хотя бы одна практическая работа по теме
- код можно спокойно объяснить своими словами
- нет критичных ошибок линтера и явных `any` там, где можно обойтись без них
- сделан короткий рефакторинг или повторное улучшение решения
- написан короткий конспект: что было новым, что оказалось сложным, что хочется повторить

## Отдельный блок: tooling и delivery

Эту тему можно проходить не строго одним модулем, а параллельно основному плану.

### Что стоит освоить

- `npm` и `pnpm`
- scripts: `dev`, `build`, `preview`, `lint`, `test`
- aliases в `Vite`
- работа с `env`
- базовый CI mindset: проект должен стабильно собираться
- деплой на `Vercel`, `Netlify` или аналогичную платформу

### Практика

- настроить scripts одинаково в нескольких проектах
- добавить проверку `lint + build` перед публикацией
- сделать deploy хотя бы для 1-2 учебных приложений

### Результат

- проект не только написан, но и доведен до состояния, близкого к реальной поставке

## Идеи для pet-project

Ниже собраны идеи, которые удобно брать в зависимости от текущего этапа. Лучше выбрать 1-2 проекта и постепенно усложнять их, чем начинать много разных и не доводить до конца.

**Уровни сложности:**

- `easy` — можно закрыть за 1-3 дня, минимум зависимостей
- `medium` — 1-2 недели, несколько страниц и слоев логики
- `hard` — 3+ недели, много сценариев, архитектура и тесты обязательны

### Как выбирать проект

- для ранних модулей лучше брать `easy` CRUD без сложной бизнес-логики
- для середины плана важны router, store, API и формы — смотри `medium`
- для поздних модулей полезно брать `hard`, где есть смысл в архитектуре, тестах и `Nuxt`
- если проект кажется слишком большим, начинай только с блока **MVP**
- блок **Потом можно добавить** — это не обязательный scope, а направление роста

### Универсальный шаблон MVP

Почти любой проект можно стартовать с такого минимума:

- список сущностей
- страница деталей
- создание или редактирование сущности
- фильтрация или поиск
- обработка `loading / error / empty`
- базовые тесты на критический сценарий

## Рекомендации по темпу

Если идти глубоко, а не быстро, разумный ориентир:

- `4-6 недель` на фундамент Vue
- `3-5 недель` на Router, Pinia, API
- `2-4 недели` на формы, тестирование и архитектуру
- `2-4 недели` на Nuxt и SSR
- `3-6 недель` на финальный проект

Итого полный цикл может занять `3-5 месяцев` при регулярной практике.

## Важная оговорка про `Options API`

Основной путь этого плана - `Composition API` и современный стек вокруг него. `Options API` стоит знать на уровне чтения чужого и legacy-кода, но не обязательно строить на нем весь учебный маршрут.

## Главное правило

Не пытайтесь просто "пройти темы". Цель этого плана — дойти до состояния, в котором вы можете самостоятельно спроектировать, реализовать, протестировать и улучшить полноценное приложение на современном `Vue`.
