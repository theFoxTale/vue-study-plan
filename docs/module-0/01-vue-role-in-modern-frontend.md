# Module 0 · Теория: роль Vue в современном frontend

Этот материал закрывает первый теоретический пункт `Module 0`: понять, **что такое Vue**, **где он находится в современном frontend-стеке** и **зачем его изучать именно сейчас**.

---

## 1. Что такое Vue

**Vue** (произносится как *view*) — JavaScript-фреймворк для построения пользовательских интерфейсов.

Он не заменяет HTML, CSS и JavaScript, а **надстраивается над ними** и предлагает:

- **декларативный** подход к UI: вы описываете, *что* должно отображаться, а не вручную меняете DOM;
- **компонентную модель**: интерфейс собирается из переиспользуемых блоков;
- **реактивность**: при изменении данных интерфейс обновляется автоматически.

Официальное определение и вводная: [Introduction · Vue.js Guide (RU)](https://ru.vuejs.org/guide/introduction.html)

---

## 2. Почему Vue называют «progressive framework»

Ключевая идея Vue — **прогрессивность** (*incrementally adoptable*):

> Vue можно использовать по-разному — от простого улучшения HTML-страницы до полноценного fullstack-приложения.

Примеры уровней использования:

| Уровень | Что это значит |
|--------|----------------|
| **HTML enhancement** | Vue подключается без сложной сборки, для небольших интерактивных блоков |
| **SPA** | Одностраничное приложение с роутингом и состоянием |
| **SSR / SSG** | Серверный или статический рендеринг через `Nuxt` |
| **Fullstack** | Frontend + server routes, API, deployment в одной экосистеме |

Vue не заставляет сразу брать весь стек. Можно начать с малого и постепенно добавлять:

- `Vue Router` — маршрутизация
- `Pinia` — глобальное состояние
- `Vite` — сборка и dev-сервер
- `Nuxt` — meta-framework для SSR/SSG и production-ready архитектуры

Это отличает Vue от подхода «возьми всё сразу или собирай экосистему сам».

---

## 3. Где Vue находится в современном frontend

Современный frontend — это не только «фреймворк для кнопок». Это целая цепочка:

```text
Browser UI
   ↓
Vue (components, reactivity, templates)
   ↓
Router / State / Forms / Data fetching
   ↓
Build tool (Vite)
   ↓
Deploy / SSR / SEO (Nuxt, hosting)
```

### Vue 3 — текущий стандарт экосистемы

- **Vue 2** завершил жизненный цикл в декабре 2023 года.
- **Vue 3** — актуальная версия: Composition API, лучшая производительность, сильная поддержка TypeScript.
- Для новых проектов изучать нужно именно **Vue 3**.

Обзор состояния экосистемы: [State of Vue.js Report 2025](https://stateofvue.framer.website/)

### Официальная экосystema вокруг Vue

| Инструмент | Роль |
|-----------|------|
| **Vue 3** | UI, реактивность, компоненты |
| **Vite** | Быстрая разработка и production-сборка |
| **Vue Router** | Навигация в SPA |
| **Pinia** | Глобальное состояние (замена Vuex) |
| **Vitest** | Unit/component-тесты |
| **Nuxt** | SSR, SSG, file-based routing, fullstack-сценарии |

Важно: эти инструменты **официально поддерживаются** и хорошо стыкуются друг с другом. Это снижает «архитектурный шум» на старте: меньше времени на выбор стека, больше — на продукт.

## 4. Для каких задач Vue особенно подходит

Vue хорошо ложится на проекты, где важны:

- **скорость разработки** и понятный код;
- **постепенное усложнение** проекта без переписывания с нуля;
- **админки, dashboards, SaaS, internal tools**;
- **контентные сайты и SEO-проекты** через `Nuxt`;
- **команды**, где нужен баланс между гибкостью и предсказуемостью.

Vue используют не только в «учебных» или маленьких проектах. Его применяют в production у крупных продуктов и команд — например, у GitLab есть публичные материалы по frontend-стеку на Vue.

## 5. Vue, React и Angular — в чем разница на практике

Не нужно выбирать «лучший фреймворк в мире». Нужно понимать **разные философии**.

| Критерий | Vue | React | Angular |
|---------|-----|-------|---------|
| **Философия** | Progressive framework, официальный стек по слоям | Библиотека UI + огромная экосистема | Полноценная platform с жесткой структурой |
| **Порог входа** | Ниже для старта с HTML/CSS/JS | Средний, много решений «сам выбирай» | Выше, больше концепций сразу |
| **Шаблоны** | HTML-подобные templates + SFC | JSX | Templates + TypeScript-first |
| **Routing / State** | Официальные решения: Router, Pinia | React Router, Redux/Zustand и др. | Встроено в платформу |
| **Meta-framework** | Nuxt | Next.js | Angular SSR / analogs |
| **Когда удобен** | Быстрый старт, зрелый официальный стек | Максимальная экосystema, много вакансий | Enterprise, строгая архитектура |

### Главная мысль

- **React** — огромная экосистема и гибкость, но много решений нужно выбирать самому.
- **Angular** — «всё из коробки», но тяжелее на входе.
- **Vue** — баланс: **простой старт + официальный стек + масштабирование до production**.

Для этого учебного плана Vue выбран потому, что он хорошо подходит для **глубокого и спокойного** изучения современного frontend без хаоса на старте.

## 6. Почему Vue актуален в 2026 году

Кратко, что важно знать сейчас:

1. **Vue 3 — стандарт**, Vue 2 больше не развивается.
2. **Composition API** — рекомендуемый стиль для новых приложений.
3. **TypeScript** стал нормой, а не «продвинутой опцией».
4. **Vite** — де-факто стандарт сборки для Vue-проектов.
5. **Pinia** — основной state manager вместо Vuex.
6. **Nuxt** — логичное продолжение для SSR, SEO и fullstack-сценариев.

Vue остаётся одним из самых практичных выборов для разработчика, который хочет:

- быстро начать писать UI;
- постепенно освоить router, store, API, forms, tests;
- дойти до production-ready приложений без смены фреймворка на полпути.

## 10. Мини-конспект

- Vue — современный UI-фреймворк для построения интерфейсов любой сложности.
- Vue **progressive**: можно начать просто и наращивать стек постепенно.
- Современный стандарт — **Vue 3 + Vite + Pinia + Vue Router + TypeScript**.
- Для SEO, SSR и fullstack логичное продолжение — **Nuxt**.
- Vue хорошо подходит для практичного и глубокого изучения frontend без лишнего хаоса на старте.
